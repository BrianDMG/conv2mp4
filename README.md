    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#conv2mp4-ps" aria-hidden="true" class="anchor" id="user-content-conv2mp4-ps"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>conv2mp4-ps</h1>
<p>Powershell script that recursively searches through a user-defined file path (<em>or paths</em>) and convert all videos of user-specified file types to <strong>MP4</strong> with <strong>H264</strong> video and <strong>AAC</strong> audio as needed using ffmpeg. If a conversion failure is detected, the script re-encodes the file with HandbrakeCLI. Upon successful encoding, Plex libraries are (optionally) refreshed and source file is deleted. The purpose of this script is to reduce the amount of transcoding CPU load on a Plex, Emby, or Kodi server and increase video compatibility across platforms.</p>
<p>Python version can be found here: <a href="https://github.com/BrianDMG/conv2mp4-py">conv2mp4-py</a></p>
